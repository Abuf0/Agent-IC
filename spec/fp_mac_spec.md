## 3. 功能描述

### 3.1 概述
本模块实现单精度浮点乘累加运算：`Z = A * B + C`，支持IEEE 754标准，包括正规数、次正规数、无穷大、NaN（Not a Number）的处理，并提供多种舍入模式。

### 3.2 算法流程
1. **输入分解**：将输入A、B、C分解为符号位、指数位、尾数位，并添加隐藏位（当指数非零时为1，否则为0）。
2. **乘法计算**：
   - 计算乘积符号：`product_sign = a_sign ^ b_sign`
   - 计算乘积指数：`product_exp = a_exp + b_exp - 127`（扣除偏置）
   - 计算乘积尾数：`product_man = {a_hidden, a_man} * {b_hidden, b_man}`（24位×24位乘法，结果48位，然后左移扩展为72位）
3. **尾数对齐**：
   - 计算指数差：`exp_diff = |product_exp - c_exp|`
   - 根据指数差对乘积尾数或C尾数进行右移对齐（最大移位48位，超过则尾数为0）
4. **加法计算**：
   - 将有符号尾数转换为二进制补码形式
   - 执行有符号加法：`sum_man = prod_val + c_val`
   - 计算绝对值：`sum_man_abs = abs(sum_man)`
   - 检测溢出：`sum_man_overflow`（当和溢出到额外位时）
5. **前导零计数（LZC）**：
   - 对72位绝对值进行前导零计数（0~72）
   - 根据LZC值判断结果类型：
     - `lzc_normal`（LZC < 48）：正规数
     - `lzc_large_subnormal`（48 ≤ LZC < 72）：次正规数
     - `lzc_huge_zero`（LZC ≥ 72）：零
6. **归一化**：
   - 正规化：尾数左移LZC位，指数减去LZC
   - 次正规化：尾数左移LZC位，指数置为零
7. **舍入处理**：
   - 根据舍入模式（io_rnd）计算舍入位（guard_bit, round_bit, sticky_bit）
   - 执行舍入，可能引起尾数溢出和指数调整
8. **结果组装**：
   - 正规数：`{result_sign, exp_final, man_final}`
   - 次正规数：`{sub_sign, 8'h00, man_final}`
   - 零：32'h00000000
9. **特殊值处理**：
   - 输入NaN：输出quiet NaN（尾数最高位为1）
   - 无穷大×零：输出NaN
   - 无穷大±无穷大（符号相反）：输出NaN
   - 无穷大：输出带正确符号的无穷大

### 3.3 舍入模式（io_rnd编码）
| io_rnd | 舍入模式 | 描述 |
|--------|----------|------|
| 3'b000 | 最近偶数（Round to nearest, ties to even） | 默认模式，符合IEEE 754标准 |
| 3'b001 | 向零舍入（Round toward zero） | 截断，不进行舍入 |
| 3'b010 | 向下舍入（Round down, toward negative infinity） | 向负无穷方向舍入 |
| 3'b011 | 向上舍入（Round up, toward positive infinity） | 向正无穷方向舍入 |
| 3'b100 | 最近远离零（Round to nearest, ties away from zero） | 平局时远离零舍入 |
| 其他 | 同3'b000 | 默认最近偶数 |

### 3.4 状态标志（io_status编码）
| io_status | 含义 | 描述 |
|-----------|------|------|
| 8'h00 | 正常 | 计算结果为正规数、次正规数或零 |
| 8'h01 | 异常NaN | 输出为NaN（包括无效操作） |
| 8'h02 | 异常无穷大 | 输出为无穷大 |

### 3.5 特殊值处理规则
- **NaN传播**：任一输入为NaN，输出NaN（quiet NaN）
- **无穷大×零**：输出NaN
- **无穷大±无穷大（符号相反）**：输出NaN
- **无穷大±有限数**：输出同号无穷大
- **零结果符号**：强制为正零（+0）

## 4. 寄存器描述
本模块为纯组合逻辑，无寄存器。

## 5. 性能指标
- **工艺节点**: TSMC 22nm
- **目标频率**: 400MHz
- **关键路径**: 预计为前导零计数（LZC）逻辑和舍入逻辑的级联
- **面积估算**: 较大，主要来源于72位前导零计数逻辑和24×24位乘法器
- **功耗预算**: 中等，由于组合逻辑复杂度较高
- **时序裕量**: 需要综合后评估，建议对关键路径进行流水线优化

## 7. 注意事项
1. **组合逻辑延迟**：本模块为纯组合逻辑，关键路径较长，建议在顶层添加流水线寄存器以满足时序要求。
2. **面积优化**：前导零计数逻辑占用较大面积，可考虑使用并行前缀树结构优化。
3. **舍入模式兼容性**：io_rnd=3'b100（最近远离零）与IEEE 754标准不完全一致，平局时向零舍入而非远离零舍入，使用时需注意。
4. **零的符号**：输出零强制为正零（+0），与标准允许的负零不一致，可能影响某些应用。
5. **状态标志简化**：io_status仅指示NaN和无穷大，未包含溢出、下溢等异常标志，需要时可在上层扩展。
6. **验证完备性**：必须对所有舍入模式和特殊值组合进行充分验证，建议使用形式验证辅助。
