# fp_mac模块验证报告

## 1. 验证环境概述

### 1.1 模块信息
- **模块名称**: fp_mac (浮点乘累加单元)
- **功能**: 单精度浮点运算 A*B+C
- **流水线**: 5级 (stage1-stage4 + 顶层包装)
- **舍入模式**: 5种 (000-最近偶数, 001-向零, 010-向下, 011-向上, 100-最近远离零)
- **输入**: io_a[31:0], io_b[31:0], io_c[31:0], io_rnd[2:0], clk, resetn
- **输出**: io_z[31:0], io_status[7:0]

### 1.2 验证平台
- **测试框架**: Cocotb (Python)
- **仿真器**: Icarus Verilog (默认)
- **仿真脚本**: Makefile + Python运行脚本
- **波形格式**: VCD/FST (可选)
- **参考模型**: Python实现的IEEE 754单精度浮点模型

### 1.3 测试目录结构

## 4. 已知问题与限制

### 4.1 确认的RTL缺陷
1. **舍入逻辑错误**（高优先级）:
   - 最近偶数模式 (RND=000): 部分边界情况舍入不正确
   - 最近远离零模式 (RND=100): 部分边界情况舍入不正确
   - **影响**: 影响计算精度，不导致功能失效
   - **建议**: 修复舍入逻辑实现

2. **次要问题**:
   - 部分异常路径覆盖率不足
   - 某些极端边界条件未充分测试

### 4.2 验证环境限制
1. 参考模型使用Python双精度浮点，与RTL单精度存在微小差异
2. 随机测试无法覆盖所有可能的输入组合
3. 形式验证未集成，需要补充

## 6. 使用指南

### 6.1 快速开始
```bash
# 进入测试目录
cd test_fp_mac

# 环境自检
python env_check.py

# 运行所有测试（生成波形）
python run.py all --waves

# 生成报告
python run.py report
```

### 6.2 详细测试执行
```bash
# 使用Makefile直接运行
make WAVES=1
make report

# 清理生成文件
make clean
```

### 6.3 添加新测试
1. 在 `test_cases.py` 中添加新的 `TestCase` 对象
2. 或在 `fp_mac_tb.py` 中添加新的 `@cocotb.test()` 函数
3. 运行测试验证新功能
